Dispense Shelves tab

Live graphical view of bottle slots on shelves F1–F4 (read-only).

Tab purpose

The Dispense Shelves tab shows four shelf layouts (F1–F4) in a 2×2 grid. Each shelf is drawn to match the physical slot arrangement (21 positions). The UI subscribes to MQTT and updates about every 200 ms while connected.

MQTT topics

The app subscribes to:

  • plc/dispense/f1 … plc/dispense/f4
  • plc/dispense (optional bundled payload with f1–f4 objects)

Topics may include a machine prefix (e.g. kadeya-001/plc/dispense/f1); the suffix is matched automatically.

Payload format

Each message is a JSON object: keys are slot IDs, values are bottle IDs or -1 for empty.

F1
Keys "1" … "21" (local slots 1–21 on the drawing).
F2
Keys "22" … "42" (mapped to local slots 1–21).
F3
Keys "43" … "63".
F4
Keys "64" … "84".
-1
Empty slot: only the slot index is shown (muted).
Positive number
Occupied slot: five-digit bottle ID (e.g. 00125) in the circle; slot index below in orange.

Numeric values may appear with or without leading zeros in the broker payload; the UI normalizes them for display.

Layout

Each shelf panel (F1–F4) is a canvas: gray plate outline, white circles per slot, black sensor tabs on the appropriate edge. No commands are sent from this tab.
