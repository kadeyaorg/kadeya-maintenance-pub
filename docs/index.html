About the application

kadeya-maintenance is a cross-platform console (Windows, macOS, Linux desktop, plus a web build) that connects to the machine MQTT broker to show status (dashboard), send maintenance commands (doors, robots, valves, etc.), and inspect MQTT messages received by the UI.

On the desktop app, a newer release is offered as an in-app update (signed, installed automatically) when one is available; you can install now or later. The Washer screen can also show Hobart cloud telemetry when configured (desktop only).

Connection screen (before tabs)

Documentation
Opens the user documentation (GitHub Pages) in the default browser.
“IP : port” buttons (recent connections)
Fill the IP and port fields automatically for quick reconnect (up to 3 entries).
Connect to Machine
Attempts broker connection, subscribes the app to the required MQTT topics, saves the connection in recents, then shows the sidebar and tabs.

Header after connection

Documentation
Same behavior as on the connection screen.
Change Machine
Returns to the connection screen (MQTT disconnect, UI reset).
Disconnect
Asks for confirmation, then disconnects and returns to the connection screen.

Sidebar (tabs)

Each entry opens the matching tab. Individual pages describe the controls for each tab:

  • Dashboard — overview and alerts
  • PLC Reset — PLC reset flags and maintenance mode
  • Doors — door open / close
  • Robots — Return and Dispense arms
  • Move Sequencer — build and play robot move sequences (file import/export)
  • Robot View — live SCARA pose visualization
  • Valves — solenoids
  • Dispense — test recipes
  • Dispense Shelves — live bottle layout (F1–F4)
  • Washer — bottle washer + Hobart cloud telemetry
  • Syrup — BIB levels
  • Software — updates and restarts
  • HMI — maintenance, fullscreen mode, unstick dispense/return flows
  • MQTT Monitor — message log
