Onglet MQTT Monitor

Journal en temps quasi réel des messages MQTT auxquels l’interface est abonnée (lecture / filtrage).

Champs de filtre

Topic
Filtre les lignes affichées : seuls les messages dont le sujet contient le texte saisi (sans tenir compte de la casse) restent visibles. Laisser vide pour tout afficher.
Search
Filtre sur une chaîne libre (horodatage, sujet ou charge utile), insensible à la casse.

Contrôles

Pause (case à cocher)
Lorsqu’elle est cochée, l’affichage ne se met plus à jour avec les nouveaux messages (le tampon continue de recevoir en arrière-plan jusqu’à la limite interne, au-delà de laquelle des messages peuvent être ignorés si le canal est saturé).
Clear
Vide le tampon et le journal affiché ; remet le compteur de messages à zéro.

Zone de texte

Chaque ligne correspond à un message : horodatage, sujet (mise en forme visuelle particulière dans l’UI Windows) et charge utile JSON compactée sur une ligne lorsque c’est du JSON valide.

Le compteur à droite indique le nombre de messages conservés dans le tampon (plafonné, les plus anciens sont retirés quand la limite est dépassée).
